# Game-Programming

상용 엔진 없이(SDL2 + 자체 물리) 2D 런게임을 만드는 최소 골격입니다.

## Build

### Windows (Visual Studio)

```bash
cmake -S . -B build -G "Visual Studio 17 2022" -A x64
cmake --build build --config Debug
```

실행 파일은 보통 `build/Debug/chrono_rush_demo.exe`에 생성됩니다.

### Windows (Ninja)

```bash
cmake -S . -B build -G Ninja
cmake --build build
```

## Controls

- `마우스 드래그`: 조준
- `X`: 폭탄 던지기
- `C`: 점프
- `Z`: 시간 역행(3초 전으로)
- `Esc`: 종료

## Repo structure

- `include/`: 헤더(수학/물리/리와인드/게임)
- `src/`: 구현
  - `app/`: 앱/윈도우/렌더 루프
  - `physics/`: 고정 timestep 물리 + 충돌(간단)
  - `rewind/`: 원형 버퍼 기반 리와인드
  - `game/`: 런게임 규칙/스폰(스켈레톤)

## Roles (팀원 A / B 분담: 옵션 1)

분량을 최대한 비슷하게 맞추기 위해, **A는 “핵심 재미(폭탄/중력장) + 객체별 스냅샷 구현”**, **B는 “러너 시스템 + 리와인드 엔진(버퍼/적용)”**을 담당합니다.

### 팀원 A: 폭탄/중력장 + 객체별 `Save/Load`(스냅샷) 구현

- **폭탄 투척/물리**
  - 마우스 조준 + `X` 투척 파워/각도 튜닝
  - 바운스/반발/마찰, 폭발 트리거(시간/충돌/조건)
- **중력장(인력/척력)**
  - 힘 모델(거리², 클램프, 범위/지속시간) 및 튜닝
  - 적용 대상/룰(상자/적/장애물), 시각 피드백(범위/이펙트) + 디버그 표시
- **스냅샷(역행 호환) — “객체별 구현”**
  - 각 객체가 저장해야 할 값 정의(예: `active, pos, vel` + 객체별 상태)
  - 각 객체에 `SaveSnapshot()` / `LoadSnapshot()`(또는 동등 함수) 구현
  - 폭발/파편/상태변화가 역행 시 어떻게 복원되는지 규칙 포함

### 팀원 B: 러너 시스템 + 리와인드 버퍼/적용 로직

- **러너(스크롤/카메라/스폰)**
  - 스폰 매니저, 오브젝트 풀링(재활용), 난이도 곡선(속도/스폰 밀도/패턴)
- **시간 역행 시스템 — “프레임 저장/복원 엔진”**
  - `RewindBuffer` 관리(3초 고정, 객체 수 변동 대응)
  - 매 프레임 “전체 객체 스냅샷 수집” → 역행 시 “전체 객체에 복원 적용”
  - 스태미나/쿨다운, 게임 상태(오버/재시작) 연동
- **UI/상태**
  - HUD(스태미나/거리/디버그 토글), 게임오버/재시작/일시정지

### 사전 합의(필수 3가지)

- **스냅샷 공통 포맷**: 최소 `active, pos, vel` + 객체별 추가 필드 목록
- **객체 ID 정책**: 역행 동안 동일 객체를 식별하는 키(풀링/재활용 고려)
- **삭제/풀링/폭발의 역행 규칙**: “되감으면 다시 살아나는가?” 기준을 1줄로 확정
