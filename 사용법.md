# 깃허브 기본 사용법 요약

### 1. 새로운 브랜치 만들고 작업 올리기
나만의 작업 공간(브랜치)을 만들고 깃허브에 처음 올릴 때 사용하는 순서입니다.

1. 새 브랜치를 만들고 바로 그곳으로 이동합니다.
   `git checkout -b [새 브랜치 이름]`
2. 방금 만든 새 브랜치를 깃허브 서버에도 등록합니다. (새 브랜치 생성 후 처음 한 번만 입력)
   `git push -u origin [새 브랜치 이름]`
3. 수정한 파일들을 저장할 준비를 합니다. (점 `.` 앞에 띄어쓰기 필수)
   `git add .`
4. 어떤 작업을 했는지 메모와 함께 컴퓨터에 저장(커밋)합니다.
   `git commit -m "[메모]"`
5. 저장된 내용을 깃허브로 현재 브랜치에 최종 업로드(푸시)합니다.
   `git push origin [새 브랜치 이름]`

Ex. '현진' 브랜치 생성 후 '사용법' 파일 올리기
   `git checkout -b 현진`
   `git push -u origin 현진`
   `git add .`
   `git commit -m "사용법 파일 올리기"`
   `git push origin 현진`

---

### 2. 기존 브랜치에 작업 올리기 (기본 3단 콤보)
1. 파일 수정하기
2. 파일 확실히 저장하기 (Ctrl + S)
3. Git 명령어 순서대로 입력하기
   `git add .`
   `git commit -m "[메모]"`
   `git push origin [현재 브랜치 이름]`

---

### 3. 브랜치 업데이트 및 이동

1. 현재 브랜치의 최신 상태 업데이트 (가져오기)
   `git pull origin [현재 브랜치 이름]`
2. 현재 브랜치의 파일은 유지하되, 다른 브랜치의 모든 파일과 코드를 가져오고 싶을 때
   1) `git pull origin [가져올 브랜치 이름]`
   2) (메모장 창이 뜰 경우) Esc -> :wq -> Enter
3. 다른 브랜치로 화면(작업 공간)을 전환하고 싶을 때
   `git checkout [이동할 브랜치 이름]`

Ex. '현진' 브랜치에서 '주형' 브랜치의 내용을 가져와서 합치고 싶을 때
   `git pull origin 주형`

---

### 4. 기존 파일 삭제 / 이름 변경하고 반영하기

1. 에디터에서 파일 지우기(Delete) 또는 이름 바꾸기(Rename/F2)
2. 파일 변경 후 터미널에 3단 콤보 입력
   `git add .`
   `git commit -m "파일 삭제 또는 이름 변경"`
   `git push origin [현재 브랜치 이름]`

---

### 5. 다른 브랜치에서 특정 파일 하나만 가져오기
   `git checkout [가져올 브랜치 이름] -- [가져올 파일 이름]`

Ex. 'main' 브랜치에서 '사용법.md' 파일을 가져오고 싶을 때
   `git checkout main -- 사용법.md`

---

### 6. 두 브랜치를 완전히 합치기 (Merge)
내 작업물을 다른 브랜치(예: main)에 밀어 넣어서 업데이트하고 싶을 때

1. 대상 브랜치(합쳐질 곳)로 이동하기
   `git checkout main`
2. 내 원래 브랜치(가져올 곳)의 내용 병합하기
   `git merge 현진`
   (메모장 화면이 뜬다면: Esc -> :wq -> Enter)
3. 합쳐진 내용을 깃허브 서버에 최종 업로드하기
   `git push origin main`

---

### 7. 긴급 복구 명령어 (주의)
실수로 수정한 파일을 가장 최근 저장 상태로 되돌리기
   `git restore [파일 이름]`
